NAT: Translates private IPs to public IPs for internet access.

TTL: Prevents infinite loops by limiting hop count.

🚦 OSI Layer 4: Transport Layer
1. Core Responsibility
End‑to‑end communication between applications.

Ensures data is delivered reliably, in order, and without duplication (TCP) or fast but without guarantees (UDP).

Adds ports to identify which application/service should receive the data.

PDU: Segment (TCP/UDP)

2. Functions of Transport Layer
Segmentation → Breaks large data into smaller segments.

Reassembly → Puts segments back together at destination.

Error detection → Checksums in TCP/UDP headers.

Flow control → Prevents sender from overwhelming receiver.

Congestion control → TCP algorithms (slow start, AIMD).

Multiplexing/demultiplexing → Uses ports to deliver data to correct application.

3. Ports
Ports identify applications/services.

Well‑known ports (0–1023): Reserved for common services.

Registered ports (1024–49151): Assigned to specific applications.

Dynamic/private ports (49152–65535): Used temporarily by clients.

Examples:

20/21 → FTP

22 → SSH

23 → Telnet

25 → SMTP

53 → DNS

80 → HTTP

443 → HTTPS

110 → POP3

143 → IMAP

3389 → RDP

4. Protocols at Layer 4
TCP (Transmission Control Protocol)

Reliable, connection‑oriented.

Features: 3‑way handshake, sequencing, acknowledgments, retransmission, flow control, congestion control.

UDP (User Datagram Protocol)

Unreliable, connectionless.

Lightweight, faster, used for streaming, DNS, VoIP.

SCTP (Stream Control Transmission Protocol)

Hybrid of TCP/UDP.

Supports multi‑homing, multi‑streaming.

5. TCP in Detail
Connection establishment → 3‑way handshake (SYN, SYN‑ACK, ACK).

Connection termination → 4‑way handshake (FIN, ACK).

Reliability → Sequence numbers + acknowledgments.

Flow control → Sliding window mechanism.

Congestion control → Algorithms like slow start, congestion avoidance.

6. UDP in Detail
No handshake → Just sends datagrams.

No reliability → No retransmission or sequencing.

Use cases → DNS queries, VoIP, video streaming, gaming.

7. Multiplexing & Demultiplexing
Multiplexing → Multiple applications share one network connection.

Demultiplexing → Receiver uses port numbers to deliver data to correct application.

8. Real‑World Flow (PC → Google.com over HTTPS)

Transport layer adds TCP header:

Src Port: random (e.g., 49160)

Dst Port: 443 (HTTPS)

TCP ensures reliable delivery with acknowledgments.

Segments passed to Layer 3 (IP packets).

At destination, TCP reassembles segments and delivers to HTTPS service.

🏠 Interior Gateway Protocols (IGPs)
1. RIP (Routing Information Protocol)
Type: Distance‑vector.

Metric: Hop count (max 15 hops → anything beyond is unreachable).

Updates: Periodic (every 30 seconds).

Advantages: Simple, easy to configure.

Disadvantages: Slow convergence, limited scalability, prone to routing loops.

Use case: Small networks, legacy systems.

2. OSPF (Open Shortest Path First)
Type: Link‑state.

Metric: Cost (based on bandwidth).

Updates: Triggered (LSAs when topology changes).

Features:

Hierarchical design (areas, backbone area 0).

Fast convergence.

Supports VLSM and CIDR.

Authentication for security.

Advantages: Scales well, widely used in enterprises.

Disadvantages: More complex to configure.

Use case: Large enterprise networks, multi‑area designs.

3. EIGRP (Enhanced Interior Gateway Routing Protocol)
Type: Hybrid (distance‑vector + link‑state features).

Metric: Composite (bandwidth, delay, reliability, load).

Updates: Triggered, uses DUAL algorithm for loop‑free paths.

Features:

Fast convergence.

Unequal‑cost load balancing.

Supports VLSM and CIDR.

Advantages: Efficient, reliable, Cisco‑optimized.

Disadvantages: Proprietary (though later partially opened).

Use case: Cisco‑based enterprise networks.

4. IS‑IS (Intermediate System to Intermediate System)
Type: Link‑state.

Metric: Cost.

Updates: Triggered (LSPs).

Features:

Similar to OSPF but used more in service provider networks.

Scales extremely well.

Flexible for IPv4 and IPv6.

Advantages: High scalability, stable in large topologies.

Disadvantages: Less common in enterprises, steeper learning curve.

Use case: ISP and carrier‑grade networks.

🌍 Exterior Gateway Protocol (EGP)
5. BGP (Border Gateway Protocol)
Type: Path‑vector.

Metric: AS path length (plus attributes like local preference, MED, weight).

Updates: Incremental, policy‑based.

Features:

Internet backbone protocol.

Supports route filtering and policy control.

Handles massive routing tables (hundreds of thousands of prefixes).

Advantages: Scales globally, highly flexible.

Disadvantages: Complex, slower convergence.

Use case: ISP interconnections, Internet routing.

🔥 Firewalls in Networking
1. Definition
A firewall is a security device or software that monitors and controls incoming and outgoing network traffic based on predefined rules.
It acts as a barrier between a trusted internal network and untrusted external networks (like the Internet).

2. Types of Firewalls
Packet‑filtering firewall

Operates at Layer 3 (IP) and Layer 4 (TCP/UDP).

Examines source/destination IP, protocol, and port numbers.

Simple, fast, but limited (doesn’t inspect payload).

Stateful inspection firewall

Tracks the state of connections (e.g., TCP handshake).

Allows only packets that are part of valid sessions.

More secure than packet‑filtering.

Application‑layer firewall

Operates at Layer 7 (Application).

Inspects traffic for specific applications (HTTP, DNS, FTP).

Can block malicious requests (e.g., SQL injection attempts).

Next‑Generation Firewall (NGFW)

Combines traditional firewall with deep packet inspection, intrusion prevention, and application awareness.

Can enforce policies based on users, applications, and content.

3. Firewall Rules
Allow/Deny based on:

Source IP

Destination IP

Protocol (TCP, UDP, ICMP)

Port number (e.g., allow TCP 443 for HTTPS, block TCP 23 for Telnet)

4. Ports & Protocols in Firewalls
Firewalls often control access to well‑known ports:

Allow TCP 80/443 → Web traffic (HTTP/HTTPS).

Allow UDP 53 → DNS queries.

Block TCP 23 → Telnet (insecure).

Allow TCP 22 → SSH (secure remote login).

They can also block/allow protocols: ICMP (ping), GRE (VPN tunneling), ESP/AH (IPsec).

5. Deployment Modes
Network‑based firewall → Hardware/software at network perimeter.

Host‑based firewall → Software running on individual machines.

Cloud firewalls → Security services provided in cloud environments.

6. Real‑World Example
Enterprise firewall rule set:

Allow TCP 443 (HTTPS) outbound to Internet.

Allow TCP 22 (SSH) inbound from admin subnet only.

Block all inbound traffic by default.

Allow ICMP echo requests internally, block externally.

🔑 Core Characteristics of EIGRP
Type: Hybrid (distance-vector + link-state features).

Protocol Number: 88.

Communication: Multicast (224.0.0.10) and unicast.

Administrative Distance (AD):

Summary routes → 5

Internal routes → 90

External routes → 170

📡 EIGRP Messages
Hello → Neighbor discovery, keep-alives (every 5 seconds).

Update → Full updates when adjacency forms, partial updates for changes.

Query → Sent when a destination becomes unreachable.

Reply → Response to queries with alternate routes.

Acknowledgement → Confirms receipt of reliable messages.

⚙️ Composite Metric
EIGRP uses a composite metric based on five K-values:

K1 → Bandwidth

K2 → Load

K3 → Delay

K4 → Reliability

K5 → MTU

Default: K1=1, K3=1, others=0 → metric depends on bandwidth + delay.

🔄 Neighbor Adjacency Requirements
For two routers to form an EIGRP neighbor relationship:

Same Autonomous System (AS) number.

Matching K-values.

Identical subnet mask.

Authentication (if enabled) must match (supports MD5 authentication).

🧮 DUAL Algorithm
Diffusing Update Algorithm (DUAL) ensures loop-free paths and fast convergence.

Maintains:

Neighbor Table → Directly connected neighbors.

Topology Table → All learned routes.

Routing Table → Best feasible successor routes.

⏱️ Timers
Hello Timer → 5 seconds.

Hold Timer (Dead Timer) → 15 seconds (neighbor declared dead if no hello received).

📌 Features
Fast convergence → Near-instant in stable networks.

Low bandwidth usage → Only sends incremental updates.

Supports VLSM & CIDR → Efficient IP addressing.

Unequal-cost load balancing → Can use multiple paths with different metrics.

Route summarization → Manual and automatic.

Redistribution → Can redistribute routes from other protocols (OSPF, RIP, static).

🏷️ Use Cases
Enterprise networks with Cisco infrastructure.

Scalable designs needing fast convergence and efficient bandwidth usage.

Networks requiring unequal-cost load balancing for redundancy.

EIGRP is a Cisco hybrid routing protocol that uses the DUAL algorithm, composite metrics, and efficient message exchange to provide fast, scalable, and reliable routing within an autonomous system.
